# Load a `s32` at byte offset 32

When you access a struct field, the compiler emits a single load instruction. Two things determine which instruction:

- **Type** → chooses the opcode (`lwz` for 32-bit word, `lbz` for byte, etc.)
- **Byte offset** → becomes the displacement in the instruction

Consider this struct and accessor:

```c
typedef struct { u8 _pad[20]; s32 val; } T;
s32 get_val(T* p) { return p->val; }
```

MWCC produces:

```asm
get_val:
  lwz  r3,20(r3)
  blr
```

`val` is a `s32` (word) at byte offset 20, so the instruction is `lwz` with displacement `20`. The layout of `S` in your exercise is different — check the displacement in the target assembly to find out where the field sits.

## Your task
Write `read` to reproduce the assembly above.

<!-- starter -->
```c
s32 read(S* s) {
    return 0;
}
```

<!-- solution -->
```c
s32 read(S* s) {
    return s->field;
}
```

<!-- context -->
