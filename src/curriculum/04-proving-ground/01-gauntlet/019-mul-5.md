# Strength reduction: non-power-of-two multipliers

The optimizer never emits a register `mullw` when multiplying by a compile-time constant. When the constant is not an exact power of two, MWCC instead uses `mulli` — multiply low immediate — which encodes the constant directly in the instruction word.

For example, multiplying by 6 becomes:

```
mulli   r3,r3,6
blr
```

The immediate field in `mulli` carries the constant. Read it from the target assembly to recover the multiplier the source used.

## Your task
Write `scale` to match the target.

<!-- starter -->
```c
int scale(int x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
int scale(int x) {
    return x * 5;
}
```
