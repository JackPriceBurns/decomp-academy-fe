# Arithmetic shift right by 14

When the compiler shifts a *signed* integer right by a constant amount it emits
**`srawi`** — the arithmetic (sign-extending) form — rather than `srwi`, which
zero-fills. The distinction is purely about signedness: the same `>>` operator
on an unsigned type produces `srwi`, while a signed type produces `srawi`.

For example, a signed right-shift by 3:

```
srawi   r3,r3,3
blr
```

The immediate field in `srawi` encodes the shift distance. Read the immediate
from the target assembly to determine how far to shift.

## Your task
Write `sar` on an `s32` so it compiles to the `srawi` above.

<!-- starter -->
```c
s32 sar(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sar(s32 x) {
    return x >> 14;
}
```
