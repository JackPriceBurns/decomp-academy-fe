# Load a `s16` at byte offset 16

When a function returns a struct field, the compiler emits a single load instruction. The load opcode depends on the field's type, and the displacement encodes its byte offset within the struct.

A `s16` field uses **`lha`** — "load halfword algebraic" (sign-extending), as opposed to `lhz` which zero-extends. Consider a different struct:

```c
typedef struct { u8 _pad[6]; s16 delta; } Packet;

s16 getDelta(Packet* p) {
    return p->delta;
}
```

That compiles to:

```asm
getDelta:
  lha r3,6(r3)
  blr
```

The displacement `6` comes from the 6-byte pad before `delta`. The signed type `s16` forces `lha` instead of `lhz`.

Look at the assembly shown above for `read`. The displacement tells you where `field` sits in `S`.

## Your task
Write `read` to reproduce the assembly above.

<!-- starter -->
```c
s16 read(S* s) {
    return 0;
}
```

<!-- solution -->
```c
s16 read(S* s) {
    return s->field;
}
```
