# Load a `u16` at byte offset 24

When a function returns a struct field, the compiler emits a single load instruction. The load opcode depends on the field's type, and the displacement encodes its byte offset within the struct.

A `u16` field uses **`lhz`** — "load halfword and zero-extend". Consider a different struct:

```c
typedef struct { u8 _pad[8]; u16 count; } Header;

u16 getCount(Header* h) {
    return h->count;
}
```

That compiles to:

```asm
getCount:
  lhz r3,8(r3)
  blr
```

The displacement `8` comes from the 8-byte pad before `count`. The type `u16` forces `lhz` rather than `lwz` or `lbz`.

Look at the assembly shown above for `read`. The displacement tells you where `field` lives in `S`; the `S` definition in the starter gives you the struct layout to match.

## Your task
Write `read` to reproduce the assembly above.

<!-- starter -->
```c
u16 read(S* s) {
    return 0;
}
```

<!-- solution -->
```c
u16 read(S* s) {
    return s->field;
}
```
