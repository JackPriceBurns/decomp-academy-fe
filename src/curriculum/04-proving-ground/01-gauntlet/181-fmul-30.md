# Scale a float

When you multiply an `f32` by a float literal, the compiler loads the constant
from the small-data area with **`lfs`**, then performs the multiply with the
single-precision **`fmuls`** instruction. Keeping both operands as `f32` avoids
a `frsp` rounding step that would appear if a double-precision intermediate were
introduced.

For example, `amplify(f32 x) { return x * 4.0f; }` produces:

```asm
amplify:
  lfs    f0, @4.0f(r2)
  fmuls  f1, f0, f1
  blr
```

Look at the assembly above: one `lfs` loads a constant, then `fmuls` uses it.
Read the constant from the small-data reference in the target assembly to
determine what literal the source used.

## Your task
Write `scale` so it compiles to the `fmuls` above.

<!-- starter -->
```c
f32 scale(f32 x) {
    return 0.0f;
}
```

<!-- solution -->
```c
f32 scale(f32 x) {
    return x * 3.0f;
}
```
