# Strength reduction: power-of-two multipliers

The optimizer never emits a register `mullw` when multiplying by a compile-time constant. When the constant is an exact power of two, it replaces the multiply with a logical left shift (`slwi`). The shift amount is the base-2 logarithm of the constant.

For example, multiplying by 8 (2³) becomes:

```
slwi    r3,r3,3
blr
```

The shift amount in the instruction encodes which power of two was used. Read the shift amount to work backwards to the original constant.

## Your task
Write `scale` to match the target.

<!-- starter -->
```c
int scale(int x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
int scale(int x) {
    return x * 2;
}
```
