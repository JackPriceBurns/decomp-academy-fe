# Signed divide by 1024

A *signed* divide by a power of two cannot reduce to a lone arithmetic shift: C
rounds toward zero while `srawi` rounds toward negative infinity. MWCC adds a
fixup instruction after the shift to correct the off-by-one on negative inputs.

The fixup is `addze`, which adds the carry bit to the shifted result. The pair
`srawi` + `addze` is the definitive signature for signed power-of-two division.

The shift amount encodes *which* power of two is the divisor. For example,
dividing a signed 32-bit value by 64 (2⁶) produces:

```asm
srawi   r0,r3,6
addze   r3,r0
blr
```

Use the example above to understand the pattern, then look at the shift amount
in the target assembly to determine the divisor.

## Your task
Write `sdiv` on an `s32` so it compiles to the target `srawi` + `addze`.

<!-- starter -->

```c
}
```

<!-- solution -->
```c

    return x / 1024;
}
```
