# Arithmetic shift right by 1

The PowerPC instruction set has two distinct right-shift opcodes:

- **`srwi`** — logical shift, zero-fills vacated high bits (used for unsigned types)
- **`srawi`** — arithmetic shift, sign-extends vacated high bits (used for signed types)

The **type** of the operand — not the `>>` operator itself — determines which
instruction the compiler emits. The immediate field encodes the shift count.

As a reference, shifting an `s32` by 3 compiles to:

```c
s32 halve(s32 x) {
    return x >> 3;
}
```

```
halve:
  srawi   r3,r3,3
  blr
```

Look at the instruction mnemonic and its immediate in the target assembly to
determine the operand type and shift amount needed.

## Your task
Write `sar` on an `s32` so it compiles to the `srawi` above.

<!-- starter -->
```c
s32 sar(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sar(s32 x) {
    return x >> 1;
}
```
