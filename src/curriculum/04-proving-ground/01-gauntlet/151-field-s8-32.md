# Load a `s8` at byte offset 32

**`lbz`** (Load Byte and Zero) loads a single byte from memory into a register, zero-extending it to 32 bits. The instruction encodes a base register and a signed displacement; the CPU reads from `base + displacement`.

Even though `s8` is a signed type, MWCC uses `lbz` rather than a sign-extending load — the ABI only requires the low 8 bits of the return register to hold a valid `s8` value, so no explicit sign extension is emitted in the callee.

For example, given:

```c
typedef struct { u8 _pad[8]; s8 status; u8 more[4]; } Node;
```

the function:

```c
s8 getStatus(Node* n) { return n->status; }
```

compiles to:

```
lbz  r3, 8(r3)
blr
```

The displacement (8) comes from the byte offset of `status` inside `Node`.

## Your task
Write `read` to reproduce the assembly above.

<!-- starter -->
```c
s8 read(S* s) {
    return 0;
}
```

<!-- solution -->
```c
s8 read(S* s) {
    return s->field;
}
```
