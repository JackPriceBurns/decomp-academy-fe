# Signed divide by 256

A *signed* divide by a power of two looks tempting to handle with a lone arithmetic shift,
but there's a rounding mismatch: C mandates truncation toward zero while `srawi` rounds
toward negative infinity. For any negative dividend that isn't already divisible by the
power of two, the shift is off by one — so MWCC emits a second instruction to correct it.

The correction is `addze`: it adds the carry flag to the shifted result, nudging the
quotient back toward zero exactly when the correction is needed. The pair is always
`srawi` then `addze`, never a lone shift.

The shift amount encodes the power of two. For a different power-of-two divisor, say 64
(2⁶), the compiler emits:

```asm
srawi   r0,r3,6
addze   r3,r0
blr
```

Use the example above to understand the pattern, then look at the shift amount in
the target assembly to determine the divisor.

## Your task
Write `sdiv` on an `s32` so it compiles to the target `srawi` + `addze`.

<!-- starter -->
```c
s32 sdiv(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sdiv(s32 x) {
    return x / 256;
}
```
