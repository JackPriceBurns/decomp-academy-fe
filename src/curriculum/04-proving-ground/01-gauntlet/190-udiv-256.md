# Unsigned divide by 256

When you divide an **unsigned** integer by a power of two, the compiler replaces the division with a logical right shift (`srwi`). There is no rounding fix-up — unsigned right-shift always rounds toward zero, which matches C's division semantics.

For example, `u32 udiv_example(u32 x) { return x / 64; }` compiles to:

```asm
srwi    r3,r3,6
blr
```

The shift amount (6) is the exponent of the divisor (2^6 = 64). To recover the divisor from an `srwi` in the wild, raise 2 to the power of the shift count.

## Your task
Write `udiv` on a `u32` so it compiles to the `srwi` above.

<!-- starter -->
```c
u32 udiv(u32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
u32 udiv(u32 x) {
    return x / 256;
}
```
