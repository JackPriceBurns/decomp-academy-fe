# Logical shift right by 11

When the compiler sees a right-shift on an **unsigned** type, it emits `srwi`
("shift right word immediate"). This is a logical shift: vacated high bits are
filled with **zero** regardless of the original value. Had the operand been
*signed*, the compiler would pick `srawi` to sign-extend instead.

As a reference, shifting a `u32` by 8 looks like:

```c
u32 extract_upper(u32 x) {
    return x >> 8;
}
```

```
extract_upper:
  srwi    r3,r3,8
  blr
```

The immediate in `srwi` encodes the shift count. Look at the immediate in the
target assembly to determine how many positions are shifted.

## Your task
Write `shr` on a `u32` to match the target.

<!-- starter -->
```c
u32 shr(u32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
u32 shr(u32 x) {
    return x >> 11;
}
```
