# Signed divide by 4

A *signed* divide by a power of two can't be a plain shift. C requires rounding toward zero, but an arithmetic right shift (`srawi`) rounds toward negative infinity for negative inputs. MWCC compensates with an `addze` instruction that nudges the result back toward zero when the remainder is non-zero.

For example, `s32 sdiv_example(s32 x) { return x / 64; }` compiles to:

```asm
srawi   r0,r3,6
addze   r3,r0
blr
```

The shift amount in `srawi` encodes the exponent: shift by 6 means divide by 2^6. The `addze` is always present for signed power-of-two division. Look at the shift count in your target assembly to determine what divisor was used.

## Your task
Write `sdiv` on an `s32` so it compiles to the `srawi` + `addze` above.

<!-- starter -->
```c
s32 sdiv(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sdiv(s32 x) {
    return x / 4;
}
```
