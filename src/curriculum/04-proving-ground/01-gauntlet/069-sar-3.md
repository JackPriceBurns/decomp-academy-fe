# Arithmetic shift right by 3

PowerPC has two flavors of right shift. `srwi` (logical) fills vacated bits
with zeroes — correct for unsigned values. `srawi` (arithmetic) replicates the
sign bit into every vacated position, keeping negative numbers negative. MWCC
chooses between them based on the *type* of the value, not the operator.

For example, a function that divides a signed integer by 2:

```
srawi   r3,r3,1
blr
```

The immediate field in `srawi` is the shift count; the output register gets the
sign-extended result. An unsigned counterpart would emit `srwi` instead.

## Your task
Write `sar` on an `s32` to reproduce the assembly above.

<!-- starter -->
```c
s32 sar(s32 x) {
    // your code here
    return 0;
}
```

<!-- solution -->
```c
s32 sar(s32 x) {
    return x >> 3;
}
```
