# Reading a three-operand instruction

PowerPC arithmetic is **three-operand**: `add rD, rA, rB` computes
`rD = rA + rB`. The destination register comes **first**, then the two source
registers. Keep that order in mind — it's an easy one to mix up at first.

For a function with two `int` arguments, the ABI places them left-to-right in
`r3` and `r4`. Look at the target assembly and ask: what C expression over
those two arguments would produce an `add` instruction using both of them?

## Your task

Write `add2`, taking two `int`s, to reproduce the target assembly.

<!-- starter -->
```c
int add2(int a, int b) {
    return 0;
}
```

<!-- solution -->
```c
int add2(int a, int b) {
    return a + b;
}
```
