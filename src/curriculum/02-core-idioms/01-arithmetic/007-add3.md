# Chaining adds (and a scheduling surprise)

Summing three integers needs two `add`s — but watch *which* registers MWCC picks.
At `-O4,p` it doesn't naïvely accumulate into `r3` left-to-right; it stashes the
first argument in a scratch register, sums the second and third first, then folds
the saved value back in. Here's how that looks for a function `sum3(x, y, z)`:

```asm
mr   r0, r3        # save x into scratch r0
add  r3, r4, r5    # y + z first
add  r3, r0, r3    # + x
blr
```

`r0` is a volatile scratch register MWCC grabs freely inside a function. (It has
one architectural quirk for later: when used as a base in a load/store address,
`r0` reads as the literal value 0 instead of its contents — so it can't be an
addressing base.)

The result is identical to a naive left-to-right sum, but the instruction order
is the optimizer's, not the source's. Predicting these small reorderings is
exactly what matching trains.

## Your task

Write `add3` to match the target.
