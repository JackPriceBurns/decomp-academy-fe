# Forcing bits on

A bitwise **OR** turns bits *on*: every bit set in the mask becomes set in the
result, and the others pass through untouched. MWCC emits the immediate form
**`ori`**.

For example, OR-ing `n` with `0x05` (decimal 5) produces:

```asm
ori  r3, r3, 5
blr
```

Unlike `andi.`, plain `ori` has **no dot** and never touches the condition
register. OR doesn't care whether the mask bits are contiguous: `ori` handles
any 16-bit pattern, so you won't see the rotate tricks that AND sometimes
triggers.

One limit to keep in mind: `ori` only carries a 16-bit immediate. A constant
wider than 16 bits splits into two instructions — `oris` for the high half-word,
then `ori` for the low — so don't be surprised to see a pair when the mask
outgrows the bottom 16 bits.

Now look at the target below. The `ori` immediate is the decimal representation
of the mask constant — convert it to find the C expression.

```asm
ori  r3, r3, 18
blr
```

## Your task

Write `set_bits` to match the target.

<!-- starter -->
```c
u32 set_bits(u32 x) {
    return 0;
}
```

<!-- solution -->
```c
u32 set_bits(u32 x) {
    return x | 0x12;
}
```
